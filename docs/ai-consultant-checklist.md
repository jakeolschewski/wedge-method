# The 120-Point AI Consulting Project Checklist

> **TL;DR:** Use this **AI consultant checklist** before, during, and after every engagement. Built from patterns across 100+ AI consulting projects. Never miss a critical step again.

---

## How to Use This AI Consultant Checklist

This checklist covers the full lifecycle of an AI consulting engagement: from first contact through final delivery and handoff. It's organized into 6 phases. Work through each phase in order. Mark items complete as you go.

Print it. Bookmark it. Add it to your project management tool. A missed checklist item at the start of a project costs 10x more to fix at the end.

---

## Phase 1: Pre-Engagement (Before You Sign)

### Discovery & Qualification (20 items)
- [ ] Confirmed budget range and decision-maker authority
- [ ] Identified the specific problem (not symptom) to be solved
- [ ] Quantified the business impact if the problem is NOT solved
- [ ] Confirmed timeline and any hard deadlines
- [ ] Identified all stakeholders who will influence or block the project
- [ ] Asked about past failed attempts to solve this problem
- [ ] Assessed the client's AI maturity (naive / aware / experimenting / deploying)
- [ ] Checked for regulatory constraints (HIPAA, GDPR, SOC 2, etc.)
- [ ] Identified the data sources you'll need access to
- [ ] Confirmed data quality and availability (the #1 AI project killer)
- [ ] Assessed internal IT security policies affecting AI tool deployment
- [ ] Identified the champion who will drive adoption internally
- [ ] Confirmed there's a realistic budget for ongoing operations, not just the project
- [ ] Checked for competing priorities that could derail the engagement
- [ ] Identified success metrics (how will we know this worked?)
- [ ] Confirmed who owns the IP and deliverables post-project
- [ ] Reviewed the client's existing tech stack for integration requirements
- [ ] Identified any change management / people challenges
- [ ] Assessed the client's tolerance for experimentation and iteration
- [ ] Ran competitive research: what are others in their industry doing with AI?

### Proposal & Contracting (10 items)
- [ ] Proposal clearly defines scope (in AND out of scope)
- [ ] Proposal includes specific, measurable deliverables
- [ ] Pricing is presented as an investment with ROI framing
- [ ] Change order process is defined in the proposal
- [ ] Contract includes IP ownership clause
- [ ] Contract includes confidentiality / NDA provisions
- [ ] Payment schedule is defined (50/50 or milestone-based)
- [ ] Contract specifies data handling and deletion policies
- [ ] Both parties have signed before any work begins
- [ ] Kickoff meeting is scheduled before contract is filed

---

## Phase 2: Kickoff & Discovery

### Project Setup (10 items)
- [ ] Kickoff meeting completed with all key stakeholders
- [ ] Project workspace created (shared drive, Slack/Teams channel, PM tool)
- [ ] Communication cadence agreed (weekly sync, async updates, escalation path)
- [ ] Access provisioned: data sources, tools, systems, APIs
- [ ] Technical environment assessed (cloud, on-prem, hybrid)
- [ ] Data access tested and confirmed (not just promised)
- [ ] Security review completed before connecting any AI tools to client data
- [ ] All tools and platforms meet client's security requirements
- [ ] Project timeline confirmed with milestones and owners
- [ ] Kickoff notes sent within 24 hours with action items and owners

### Requirements Gathering (10 items)
- [ ] Business requirements documented (what the system must DO)
- [ ] Technical requirements documented (how it must WORK)
- [ ] User requirements documented (who will use it and how)
- [ ] Non-functional requirements: performance, scale, uptime, latency
- [ ] Data requirements: sources, formats, freshness, volume
- [ ] Integration requirements: what does this connect to?
- [ ] Security requirements: authentication, authorization, audit logging
- [ ] Compliance requirements: regulations that apply to this use case
- [ ] Acceptance criteria defined: what does "done" look like?
- [ ] Requirements signed off by client champion before design begins

---

## Phase 3: Design & Architecture

### Solution Design (15 items)
- [ ] 2–3 solution options designed and evaluated (not just one)
- [ ] Trade-off analysis completed (cost, complexity, risk, time)
- [ ] Recommended approach documented with rationale
- [ ] Architecture diagram created
- [ ] Data flow diagram created (where does data go, who touches it)
- [ ] AI model selection justified (why this model for this use case)
- [ ] Prompt engineering strategy defined
- [ ] Evaluation framework defined (how will you measure AI output quality)
- [ ] Fallback/error handling strategy defined
- [ ] Human-in-the-loop requirements identified
- [ ] Cost model built (API costs, compute, storage, per-transaction)
- [ ] Security model reviewed by client IT (if applicable)
- [ ] Privacy impact assessment completed
- [ ] Build vs. buy decisions documented
- [ ] Client has approved the design before build begins

---

## Phase 4: Build & Iteration

### Development & Testing (20 items)
- [ ] Development environment set up (separate from production)
- [ ] Version control in use (git, with meaningful commit messages)
- [ ] Secrets management in place (no API keys in code)
- [ ] AI prompts versioned and documented
- [ ] Unit tests written for all custom logic
- [ ] Integration tests written for all external connections
- [ ] AI output quality tests defined and running
- [ ] Edge cases identified and handled
- [ ] Error handling and logging implemented
- [ ] Performance tested under realistic load
- [ ] Cost per transaction measured (will it be affordable at scale?)
- [ ] Security scanning completed (OWASP, dependency audit)
- [ ] Hallucination risk assessed and mitigated for the use case
- [ ] Output validation layer implemented (AI doesn't have final say on critical decisions)
- [ ] Weekly demos to client champion (no surprise deliveries)
- [ ] Feedback incorporated within 48 hours of receipt
- [ ] Change orders documented and approved before implementing scope changes
- [ ] Documentation written alongside build (not after)
- [ ] Deployment runbook drafted
- [ ] Rollback plan documented

---

## Phase 5: Delivery & Handoff

### Launch & Transition (15 items)
- [ ] User acceptance testing (UAT) completed and signed off
- [ ] Training delivered to all user groups
- [ ] Training documentation and video recordings created
- [ ] Admin/operations documentation completed
- [ ] Runbook covers: deployment, monitoring, incident response, scaling
- [ ] Monitoring and alerting configured
- [ ] Performance baseline established
- [ ] Data backup and recovery tested
- [ ] Support escalation path defined (who do they call when it breaks?)
- [ ] 30-day hypercare period agreed (you're available for rapid support)
- [ ] All access credentials transferred to client securely
- [ ] All project files and assets handed off to client
- [ ] IP transfer documentation signed (if applicable)
- [ ] Final invoice sent
- [ ] Client sign-off on project completion received in writing

---

## Phase 6: Post-Engagement

### Relationship & Business Development (20 items)
- [ ] Post-project review meeting completed (what worked, what didn't)
- [ ] Lessons learned documented for your own knowledge base
- [ ] Client satisfaction survey sent (NPS or custom)
- [ ] Testimonial or case study requested
- [ ] Referral request made (best time: when client is happiest)
- [ ] Follow-up scheduled at 30 days to check on adoption
- [ ] Follow-up scheduled at 90 days to identify next opportunities
- [ ] Client added to newsletter/CRM
- [ ] Case study drafted (with client approval)
- [ ] Project added to portfolio with metrics
- [ ] LinkedIn recommendation requested
- [ ] New opportunities identified and added to pipeline
- [ ] Rate card reviewed: was this project priced correctly?
- [ ] Scope creep documented: what was outside the SOW?
- [ ] Template updated with any new reusable components
- [ ] New prompts or AI workflows added to personal library
- [ ] Invoice paid and confirmed
- [ ] Project archived with all documentation
- [ ] Skills assessment: what did you learn? What would you do differently?
- [ ] WEDGE systems updated with new patterns and insights
